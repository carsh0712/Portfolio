# Day 8. 관리 화면과 공개 화면의 정보 노출 다듬기

## 시작하며

기능이 늘어나면 화면은 쉽게 복잡해진다. 특히 프로필은 이미지, 이메일, 소개, 링크, 유동 항목까지 담기 때문에 관리 화면 카드가 금방 무거워질 수 있다.

오늘은 새 기능을 더 추가하기보다, 이미 만든 프로필 관련 UI의 정보 노출 방식을 다듬었다.

## 오늘의 목표

- 프로필 관리 카드에서 꼭 필요한 정보만 보여준다.
- 유동 항목과 소개글은 관리 카드에서 생략한다.
- 공개 프로필 다이얼로그의 링크 색상과 아이콘을 반영한다.
- 이메일 복사 아이콘을 더 의미가 맞는 아이콘으로 교체한다.

## 프로필 카드 간소화

프로필 관리 화면의 카드는 목록에서 빠르게 구분하는 용도다. 모든 내용을 미리 보여줄 필요는 없었다.

그래서 카드에서는 다음 정보만 남겼다.

- 아바타
- 표시 이름
- 기본 프로필 여부
- 링크 개수
- 한 줄 소개
- 이메일
- 링크 이름

반대로 아래 정보는 카드에서 생략했다.

- 소개글
- 유동 항목 개수
- 유동 항목 미리보기

이 정보들은 편집 폼에서 확인할 수 있으므로, 목록 카드에는 보여주지 않는 편이 더 읽기 좋다고 판단했다.

## 공개 프로필 링크 스타일 적용

프로필 링크에는 배경색, 글자색, 아이콘을 설정할 수 있다. 하지만 공개 프로필 다이얼로그에서는 처음에 이 스타일이 적용되지 않고 기본 흰색 버튼으로만 보였다.

이를 수정해서 링크 데이터의 색상 필드를 반영하도록 했다.

```tsx
style={{
  backgroundColor: link.backgroundColor || link.background_color || undefined,
  color: link.textColor || link.text_color || undefined,
}}
```

색상이 없는 링크는 기존 기본 스타일을 유지하고, 색상이 있는 링크만 inline style로 덮어쓰게 했다.

## Copy 아이콘 적용

이메일 복사 버튼은 기존에 직접 만든 Clipboard SVG를 사용하고 있었다.

하지만 버튼의 실제 의미는 "클립보드"보다 "복사"에 가까워서 lucide의 `Copy` 아이콘으로 교체했다. 기존 컴포넌트 이름은 유지하고 내부 구현만 바꿔, 사용처를 크게 흔들지 않았다.

## 고민한 점

관리 화면과 공개 화면은 목적이 다르다.

관리 화면은 반복적으로 수정하는 사람이 빠르게 훑어보는 화면이고, 공개 화면은 방문자에게 잘 보여주는 화면이다.

그래서 관리 카드에서는 정보를 줄이고, 공개 다이얼로그에서는 링크 스타일과 아이콘을 살리는 방향을 선택했다.

## 오늘의 결과

프로필 관리 화면은 더 가벼워졌고, 공개 프로필 다이얼로그는 사용자가 설정한 링크 스타일을 더 잘 반영하게 되었다.

작은 UI 정리지만, 실제로 서비스를 사용할 때 체감되는 정돈감이 좋아졌다.
