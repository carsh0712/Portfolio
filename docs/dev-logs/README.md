# 포트폴리오 프로젝트 개발 일지

이 디렉터리는 포트폴리오 관리 서비스 개발 과정을 날짜 대신 작업 흐름 중심의 `Day` 단위로 정리한 개발 로그 모음이다.

각 문서는 그날의 목표, 구현한 내용, 왜 그렇게 설계했는지, 확인한 것, 그리고 다음 작업까지 이어지도록 기록한다.

## 목록

| Day | 제목 | 전달 주제 |
| --- | --- | --- |
| [Day 1](./day-01.md) | 포트폴리오 관리 서비스의 뼈대 만들기 | 프로젝트 구조, 클라이언트와 서버 분리, DB 방향 |
| [Day 2](./day-02.md) | 사용자 인증과 보호 라우트 구현 | 회원가입, 로그인, JWT, 보호 라우트 |
| [Day 3](./day-03.md) | 포트폴리오 관리 기능 만들기 | 포트폴리오 CRUD, 사용자별 데이터 분리 |
| [Day 4](./day-04.md) | 프로젝트 상세 정보를 다루게 만들기 | 프로젝트 CRUD, 태그, 기술 스택, 링크 |
| [Day 5](./day-05.md) | 이미지 업로드와 썸네일 처리 구현 | 파일 업로드, WebP 변환, 공개 파일 연결 |
| [Day 6](./day-06.md) | 외부에 보여줄 공개 포트폴리오 화면 만들기 | 공개 URL, 공개 API, 인터페이스 |
| [Day 7](./day-07.md) | 포트폴리오에 작성한 프로젝트 연결하기 | 다중 프로젝트, 공개 프로젝트 이동 및 목록 |
| [Day 8](./day-08.md) | 관리 화면과 공개 화면의 정보 표현 다듬기 | UI 정리, 링크 스타일, 아이콘 |
| [Day 9](./day-09.md) | 운영 DB 스키마까지 모델 맞히기 | 마이그레이션, FK, 실제 DB 비교 |
| [Day 10](./day-10.md) | 테스트로 주요 기능 안정화하기 | 프론트엔드와 백엔드 테스트, 빌드 검증 |
| [Day 11](./day-11.md) | 공개 페이지 SEO와 공유 미리보기 개선 | HTML 메타 주입, Open Graph, 카카오톡 공유 확인 |
| [Day 12](./day-12.md) | MCP 서버 개발 | 포트폴리오 API 도구화, MCP 라우팅, 구조 분리 |

## 작성 원칙

- 단순 작업 목록보다 왜 그렇게 만들었는지를 함께 남긴다.
- 코드 조각은 꼭 필요한 경우에만 짧게 넣는다.
- 문제와 해결 과정을 숨기지 않고 기록한다.
- 다음 작업으로 자연스럽게 이어질 수 있게 마무리한다.
