# 포트폴리오 프로젝트 개발 일지

이 디렉터리는 포트폴리오 관리 서비스 개발 과정을 블로그에 옮겨 적기 쉬운 형태로 정리한 개발 일지입니다.

실제 작업 흐름을 기능 단위로 재구성했습니다. 각 글은 하루 단위로 읽히도록 목표, 구현 내용, 고민한 점, 다음 작업을 포함합니다.

## 목차

| Day | 제목 | 핵심 주제 |
| --- | --- | --- |
| [Day 1](./day-01.md) | 포트폴리오 관리 서비스의 뼈대 만들기 | 프로젝트 구조, 클라이언트/서버 분리, DB 방향 |
| [Day 2](./day-02.md) | 사용자 인증과 보호 라우트 구현 | 회원가입, 로그인, JWT, 보호 라우트 |
| [Day 3](./day-03.md) | 포트폴리오 관리 기능 만들기 | 포트폴리오 CRUD, 사용자별 데이터 분리 |
| [Day 4](./day-04.md) | 프로젝트 상세 정보를 풍부하게 만들기 | 프로젝트 CRUD, 태그, 기술 스택, 링크 |
| [Day 5](./day-05.md) | 이미지 업로드와 썸네일 처리 구현 | 파일 업로드, WebP 변환, 공개 파일 접근 |
| [Day 6](./day-06.md) | 외부에 보여줄 공개 포트폴리오 화면 만들기 | 공개 URL, 공개 API, 필터링 |
| [Day 7](./day-07.md) | 포트폴리오에 작성자 프로필 연결하기 | 다중 프로필, 공개 프로필, 유동 항목 |
| [Day 8](./day-08.md) | 관리 화면과 공개 화면의 정보 노출 다듬기 | UI 정리, 링크 스타일, 아이콘 |
| [Day 9](./day-09.md) | 운영 DB 스키마와 모델 동기화 점검 | 마이그레이션, FK, 실제 DB 비교 |
| [Day 10](./day-10.md) | 테스트로 주요 기능 안정화하기 | 프론트/백엔드 테스트, 빌드 검증 |
| [Day 11](./day-11.md) | 공개 페이지 SEO와 공유 미리보기 개선 | HTML 메타 주입, Open Graph, 카카오톡 공유 확인 |

## 작성 원칙

- 단순 작업 목록보다 "왜 그렇게 만들었는지"를 함께 기록한다.
- 코드 조각은 필요한 경우에만 짧게 첨부한다.
- 문제와 해결 과정을 숨기지 않고 남긴다.
- 다음 날 작업으로 자연스럽게 이어지도록 마무리한다.
