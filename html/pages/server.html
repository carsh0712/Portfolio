서버 정보

portfolio_project_server_flask의 Flask 앱 구조, API, 모델, 환경 변수, Swagger 문서 URL을 최신 코드 기준으로 정리합니다.

요약 공통 정보 클라이언트 서버 업로드 정책 개선 상태 Note
Server

개요

역할

Flask 기반 REST API 서버입니다. 인증, 사용자, Portfolio, Project, 파일 업로드/다운로드, 공개 조회 API를 제공합니다.

주요 의존성

  • Flask: HTTP 서버와 Blueprint 라우팅
  • SQLAlchemy, pymysql: MySQL ORM과 드라이버
  • pydantic: 요청/응답 스키마 검증
  • python-jose, passlib[bcrypt]: JWT와 비밀번호 해시
  • flasgger: 개발 환경 Swagger UI
  • Pillow: 업로드 이미지 리사이즈, WebP 변환, 썸네일 생성
Runtime

실행 구조

  • app.py의 create_app()이 Flask 앱을 생성합니다.
  • 요청 시작 시 g.db = SessionLocal()로 DB 세션을 열고, 요청 종료 후 세션을 닫습니다.
  • Base.metadata.create_all(bind=engine)로 필요한 테이블을 생성합니다.
  • CORS는 core/cors.py에서 처리하며, DB의 cors_origin 테이블을 기준으로 허용 origin을 확인합니다.
  • CLIENT_DIST_DIR 또는 기본값 portfolio_project_client_vite/dist에 index.html이 있으면 /와 SPA 경로에서 빌드된 클라이언트를 제공합니다.
  • 기본 개발 실행 포트는 8000입니다.

Swagger 문서 URL

개발 환경에서 서버를 실행하면 Swagger UI는 http://localhost:8000/docs에서 확인할 수 있습니다. OpenAPI JSON 스펙은 http://localhost:8000/apispec.json입니다.

조건: Swagger UI는 APP_ENV=development일 때만 활성화됩니다.

Environment

.env 파일 작성 방법

core/config.py는 APP_ENV 값에 따라 .env.development, .env.production, .env 순서로 환경 변수를 로드합니다.

1. 서버 디렉터리로 이동

cd portfolio_project_server_flask

2. 개발 환경 파일 예시: .env.development

APP_ENV=development
SERVER_CODE=PORTFOLIO_API

MYSQL_HOST=localhost
MYSQL_PORT=64306
MYSQL_DB=portfolio
MYSQL_USER=portfolio
MYSQL_PASSWORD=change-this-password

JWT_SECRET_KEY=change-this-to-a-long-random-secret
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=30
JWT_REFRESH_TOKEN_EXPIRE_DAYS=7

UPLOAD_DIR=./uploads
CLIENT_DIST_DIR=../portfolio_project_client_vite/dist
MAX_FILE_SIZE=10485760
DB_MAX_RETRIES=2
DB_RETRY_DELAY=3

3. 운영 환경 파일 예시: .env.production

APP_ENV=production
SERVER_CODE=PORTFOLIO_API

MYSQL_HOST=your-production-db-host
MYSQL_PORT=3306
MYSQL_DB=portfolio
MYSQL_USER=portfolio
MYSQL_PASSWORD=use-a-secure-password

JWT_SECRET_KEY=use-a-long-random-production-secret
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=30
JWT_REFRESH_TOKEN_EXPIRE_DAYS=7

UPLOAD_DIR=/var/www/portfolio/uploads
CLIENT_DIST_DIR=/var/www/portfolio/client-dist
MAX_FILE_SIZE=10485760
DB_MAX_RETRIES=5
DB_RETRY_DELAY=3

4. CORS origin 등록

INSERT INTO cors_origin (server_code, origin, created_at)
VALUES ('PORTFOLIO_API', 'http://localhost:5173', NOW());
주의: 실제 실행 환경에서는 DB 비밀번호와 JWT_SECRET_KEY를 반드시 안전한 값으로 지정해야 합니다. 운영 환경에서는 Swagger UI가 꺼지도록 APP_ENV=production을 사용합니다.
Deployment

운영 배포 전 환경 체크

배포 전에 DB, JWT, CORS, 업로드 경로 설정 누락을 자동으로 확인합니다.

기본 실행

node scripts/check-production-env.mjs --cors-origin https://your-portfolio-domain.com

다른 환경 파일 검사

node scripts/check-production-env.mjs \
  --env-path portfolio_project_server_flask/.env.production \
  --cors-origin https://your-portfolio-domain.com
영역검사 항목
DBMYSQL_HOST, MYSQL_PORT, MYSQL_DB, MYSQL_USER, MYSQL_PASSWORD
JWTJWT_SECRET_KEY, JWT_ALGORITHM, access/refresh token 만료 시간
업로드/정적 파일UPLOAD_DIR 절대 경로, CLIENT_DIST_DIR 클라이언트 빌드 경로. 상대 경로는 서버 디렉터리 기준으로 해석, MAX_FILE_SIZE
DB 재시도DB_MAX_RETRIES, DB_RETRY_DELAY
CORS--cors-origin으로 운영 공개 origin 지정 여부

검사 성공 시 cors_origin 테이블에 등록할 SQL을 함께 출력합니다.

Runbook

실행과 테스트

cd portfolio_project_server_flask
pip install -r requirements.txt
python app.py
pytest

클라이언트 빌드 후 서버에서 제공

cd portfolio_project_client_vite
npm install
npm run build
cd ../portfolio_project_server_flask
python app.py

http://localhost:8000/으로 접속하면 빌드된 클라이언트가 열리고, API는 기존처럼 /api/v1/* 경로를 사용합니다.
