서버 정보

portfolio_project_server_fastapi의 Flask 서버 구조, API, 데이터 모델을 정리했습니다.

요약 공통 정보 클라이언트 서버 개선 포인트
Server

개요

역할

Flask 기반 API 서버입니다. Blueprint 라우터, SQLAlchemy ORM, Pydantic 스키마, JWT 인증, 파일 업로드/다운로드, 공개 조회 API를 제공합니다.

주요 의존성

  • Flask: HTTP 서버와 Blueprint 라우팅
  • SQLAlchemy, pymysql: MySQL ORM/드라이버
  • pydantic: 요청/응답 스키마 검증
  • python-jose, passlib[bcrypt]: JWT와 비밀번호 해싱
  • flasgger: 개발 환경 Swagger UI
Runtime

실행 구조

  • app.py의 create_app()이 Flask 앱을 생성합니다.
  • 요청 시작 전 g.db = SessionLocal()로 DB 세션을 열고, 앱 컨텍스트 종료 시 세션을 닫습니다.
  • Base.metadata.create_all(bind=engine)으로 테이블을 자동 생성합니다.
  • 개발 환경에서는 /docs에 Flasgger 기반 Swagger UI가 활성화됩니다.
  • 기본 실행 포트는 8000입니다.

Swagger 문서 URL

서버를 개발 환경으로 실행하면 Swagger UI는 http://localhost:8000/docs에서 확인할 수 있습니다. OpenAPI JSON 스펙은 http://localhost:8000/apispec.json으로 제공됩니다.

조건: Swagger UI는 APP_ENV=development일 때만 활성화됩니다. APP_ENV=production에서는 문서 UI가 등록되지 않습니다.

Environment

.env 파일 작성 방법

서버는 core/config.py에서 환경변수를 로드합니다. APP_ENV 값이 development이면 .env.development, production이면 .env.production을 우선 읽고, 해당 파일이 없으면 .env를 읽습니다.

1. 서버 디렉터리로 이동

cd portfolio_project_server_fastapi

2. 개발용 .env 파일 생성

로컬 개발 환경에서는 .env.development 파일을 만드는 방식을 권장합니다. 운영 환경 값과 개발 환경 값을 분리할 수 있기 때문입니다. 

APP_ENV=development
SERVER_CODE=PORTFOLIO_API

MYSQL_HOST=localhost
MYSQL_PORT=64306
MYSQL_DB=portfolio
MYSQL_USER=portfolio
MYSQL_PASSWORD=change-this-password

JWT_SECRET_KEY=change-this-to-a-long-random-secret
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=30
JWT_REFRESH_TOKEN_EXPIRE_DAYS=7

UPLOAD_DIR=./uploads
MAX_FILE_SIZE=10485760
DB_MAX_RETRIES=2
DB_RETRY_DELAY=3

3. 운영용 .env 파일 생성

운영 배포에서는 .env.production을 만들고 APP_ENV=production을 설정합니다. 특히 JWT_SECRET_KEY와 DB 비밀번호는 예시값을 그대로 쓰면 안 됩니다. 

APP_ENV=production
SERVER_CODE=PORTFOLIO_API

MYSQL_HOST=your-production-db-host
MYSQL_PORT=3306
MYSQL_DB=portfolio
MYSQL_USER=portfolio
MYSQL_PASSWORD=use-a-secure-password

JWT_SECRET_KEY=use-a-long-random-production-secret
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=30
JWT_REFRESH_TOKEN_EXPIRE_DAYS=7

UPLOAD_DIR=/var/www/portfolio/uploads
MAX_FILE_SIZE=10485760
DB_MAX_RETRIES=5
DB_RETRY_DELAY=3

4. CORS origin 등록

이 서버는 CORS 허용 origin을 cors_origin 테이블에서 조회합니다. 프론트엔드 개발 서버가 http://localhost:5173에서 실행된다면, 같은 SERVER_CODE 값으로 origin을 DB에 등록해야 브라우저 요청이 허용됩니다. 

INSERT INTO cors_origin (server_code, origin, created_at)
VALUES ('PORTFOLIO_API', 'http://localhost:5173', NOW());
주의: 코드에는 기본 DB 비밀번호와 fallback JWT secret이 들어 있지만, 실제 실행 환경에서는 반드시 .env 계열 파일이나 시스템 환경변수로 안전한 값을 지정해야 합니다.

실행과 테스트

cd portfolio_project_server_fastapi
pip install -r requirements.txt
python app.py
pytest

서버는 MySQL 연결을 먼저 시도합니다. MYSQL_HOST, MYSQL_PORT, MYSQL_DB, MYSQL_USER, MYSQL_PASSWORD, JWT_SECRET_KEY, UPLOAD_DIR, APP_ENV 등의 환경변수를 확인해야 합니다.
