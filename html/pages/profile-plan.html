프로필 기획

사용자가 여러 개의 자기소개 프로필을 만들고, 공개 Portfolio마다 하나의 프로필을 연결해 목적별 소개 정보를 다르게 보여주는 기능 기준입니다.

업데이트: 2026-07-07 표시 방식: 프로필 보기 버튼 + 다이얼로그 저장 방식: 핵심 컬럼 + extra_fields JSON
요약 공통 정보 클라이언트 서버 SEO AI 개발 방식 업로드 정책 프로필 기획 개선 상태 Note
Goal

기획 목적

목적별 자기소개 분리

취업용, 외주 제안용, 게임 제작자용, 아트워크용처럼 서로 다른 맥락의 자기소개를 Profile 단위로 분리합니다.

유동 항목 확장

역할, 활동 지역, 가능한 작업, 선호 기술처럼 Profile마다 달라질 수 있는 항목은 extra_fields JSON으로 관리해 코드 수정 없이 확장할 수 있게 합니다.

Storage

저장 구조 결정

컬럼으로 고정할 정보

권한, 검색, 기본값, 공개 응답에서 안정적으로 써야 하는 값은 명시 컬럼으로 둡니다.

  • display_name
  • email
  • headline
  • bio
  • avatar_file_uuid
  • is_default

JSON으로 확장할 정보

프로필 목적에 따라 개수와 이름이 바뀌는 항목은 JSON 배열로 관리합니다.

  • links: 외부 링크 목록
  • extra_fields: 라벨/값 기반 유동 항목
[
  {
    "key": "role",
    "label": "역할",
    "value": "Frontend Developer",
    "type": "text",
    "is_public": true,
    "order": 1
  },
  {
    "key": "available_for",
    "label": "가능한 작업",
    "value": "외주, 협업, 정규직",
    "type": "tag",
    "is_public": true,
    "order": 2
  }
]

Display

공개 페이지 노출 기준

1. 기본 화면 공개 Portfolio에는 제목, 설명, 필터, Project 목록을 먼저 보여줍니다.
2. 프로필 보기 버튼 연결 Profile이 있을 때만 작고 명확한 버튼을 표시합니다.
3. 다이얼로그 버튼을 누르면 Profile 다이얼로그를 열고 아바타, 이름, 소개, e-mail, 링크를 보여줍니다.
4. 닫기 사용자는 Esc, 닫기 버튼, 배경 클릭으로 다이얼로그를 닫고 Project 목록으로 즉시 돌아옵니다.
현재 결정: 초기 구현은 다이얼로그 방식으로 진행합니다. 독립 Profile 라우팅은 SEO와 공유 요구가 커질 때 확장 후보로 둡니다.
Policy

운영 정책

항목기준
생성 개수사용자 한 명은 여러 Profile을 생성할 수 있습니다. 초기 제한은 두지 않되, 운영상 필요하면 추후 상한을 둡니다.
기본 프로필사용자는 기본 Profile을 하나 지정할 수 있습니다. 첫 Profile은 자동으로 기본 Profile이 됩니다.
Portfolio 연결Portfolio 하나에는 Profile 하나만 연결합니다. 연결하지 않을 수도 있습니다.
e-mail선택 입력입니다. 공개 Profile 다이얼로그에서 mailto: 링크로 제공합니다.
유동 항목extra_fields는 key, label, value, type, is_public, order를 가진 배열로 저장합니다.
유동 항목 공개공개 다이얼로그에는 is_public=true인 항목만 order 순서대로 표시합니다.
삭제 정책Portfolio에 연결된 Profile은 바로 삭제하지 않고, 연결 해제 또는 다른 Profile로 변경한 뒤 삭제하도록 합니다.
공개 범위공개 Portfolio에 연결된 Profile만 공개 API로 노출합니다. 비공개 Portfolio의 Profile은 노출하지 않습니다.
이미지 정책아바타는 기존 이미지 업로드 정책을 따릅니다. JPG, PNG, WebP만 허용하고 GIF는 차단합니다.

UX

UI 기준

프로필 관리

  • 표시 이름, e-mail, 한 줄 소개, 상세 소개를 입력합니다.
  • 유동 항목은 키, 라벨, 값, 타입, 공개 여부, 순서로 편집합니다.
  • 아바타와 링크 편집은 기존 업로드/링크 UI를 재사용합니다.
  • 목록에는 기본 Profile badge, 링크 개수, 유동 항목 개수를 보여줍니다.

Portfolio 편집

  • 사용자의 Profile 목록을 select로 제공합니다.
  • Profile 연결 안 함을 선택할 수 있습니다.
  • 다른 사용자의 Profile ID는 서버에서 거절합니다.

공개 Portfolio

  • Profile은 기본 화면에 직접 펼치지 않습니다.
  • 프로필 보기 버튼을 눌렀을 때 다이얼로그를 엽니다.
  • 다이얼로그 안에 e-mail, 공개 유동 항목, 외부 링크를 표시합니다.
SEO

SEO 연계 기준

  • 초기에는 공개 Portfolio 페이지의 SEO를 우선합니다.
  • Profile의 display_name, headline은 Portfolio meta description 또는 Open Graph 설명 보조 정보로 활용할 수 있습니다.
  • 독립 Profile URL이 필요해지면 /public/{username}/profile/{profile_id} 형태의 라우팅을 별도 확장 후보로 검토합니다.
  • e-mail은 검색 엔진용 구조화 데이터에 바로 노출하지 않고, 공개 다이얼로그의 명시적 사용자 액션 뒤에만 제공합니다.
Validation

검증 포인트

  • 다른 사용자의 Profile ID로 Portfolio를 생성하거나 수정할 수 없어야 합니다.
  • 기본 Profile을 새로 지정하면 기존 기본 Profile은 자동으로 해제되어야 합니다.
  • 연결된 Profile 삭제는 409 응답으로 막아야 합니다.
  • 공개 Profile 아바타 파일은 공개 Portfolio에서 참조될 때만 공개 파일 API로 조회되어야 합니다.
  • GIF 파일은 아바타 업로드에서도 동일하게 차단되어야 합니다.
  • 공개 Portfolio 첫 화면에는 Profile 상세 정보가 바로 노출되지 않아야 합니다.
  • extra_fields 중 is_public=false인 항목은 공개 다이얼로그에 표시하지 않아야 합니다.
