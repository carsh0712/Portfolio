Architecture

연동 구조

브라우저가 React SPA를 실행하고 화면 라우팅을 처리합니다.
API 클라이언트인 src/utils/api.ts가 서버 API 주소로 HTTP 요청을 보냅니다.
인증 요청에는 Authorization: Bearer {access_token} 헤더가 붙습니다.
Flask 서버는 /api/v1/* Blueprint 라우터에서 요청을 처리합니다.
MySQL에는 사용자, 카테고리, 프로젝트, 파일, CORS origin, 폐기 토큰 정보가 저장됩니다.
Client SPA
  -> src/utils/api.ts
  -> http://localhost:8000/api/v1/*
  -> Flask app.py
  -> routers/*.py
  -> SQLAlchemy models.py
  -> MySQL + uploads directory
Flows

공통 동작 흐름

로그인과 토큰 갱신

  1. 사용자가 /login에서 이메일과 비밀번호를 입력합니다.
  2. 클라이언트는 POST /api/v1/auth/login을 호출합니다.
  3. 서버는 email로 사용자를 찾고 bcrypt 해시로 비밀번호를 검증합니다.
  4. 성공 시 access token과 refresh token을 반환합니다.
  5. 클라이언트는 토큰을 localStorage.auth_tokens에 저장하고 /api/v1/user/me를 호출합니다.
  6. 401 응답이 발생하면 /api/v1/auth/refresh로 access token을 재발급받아 원 요청을 재시도합니다.

파일 업로드와 이미지 표시

  1. 카테고리 대표 이미지, 프로젝트 썸네일, 스크린샷은 먼저 /api/v1/files/upload로 업로드됩니다.
  2. 서버는 파일을 UPLOAD_DIR에 저장하고 UploadFile 메타데이터를 DB에 기록합니다.
  3. 클라이언트는 응답의 uuid를 screenshot.file_uuid, thumbnail.file_uuid 같은 참조값으로 저장합니다.
  4. 관리 화면은 인증 파일 API인 /api/v1/files/{uuid}를 사용하고, 공개 화면은 /api/v1/public/{username}/file/{uuid}를 사용합니다.

공개 페이지 조회

  1. 외부 사용자는 /public/:username/:categoryCode로 접근합니다.
  2. 클라이언트는 인증 헤더 없이 공개 API를 호출합니다.
  3. 서버는 사용자, 카테고리, 프로젝트가 모두 공개 상태인지 확인한 뒤 데이터를 반환합니다.
