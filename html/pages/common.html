Architecture

연동 구조

React SPA가 화면 라우팅과 사용자 입력을 처리합니다.
API 클라이언트인 src/utils/api.ts가 서버 REST API를 호출합니다.
인증 요청에는 Authorization: Bearer {access_token} 헤더가 붙습니다.
Flask 서버는 /api/v1/* 엔드포인트로 인증, Portfolio, Project, 파일 API를 제공합니다.
DB와 uploads에는 사용자, Portfolio, Project, 파일 메타데이터와 실제 파일이 저장됩니다.
Client SPA
  -> src/utils/api.ts
  -> http://localhost:8000/api/v1/*
  -> Flask app
  -> routers / schemas / models
  -> database + uploads directory
Flows

공통 동작 흐름

로그인과 토큰 갱신

  1. 사용자가 /login에서 email과 password를 입력합니다.
  2. 클라이언트가 POST /api/v1/auth/login을 호출합니다.
  3. 성공하면 access token과 refresh token을 localStorage.auth_tokens에 저장합니다.
  4. 인증 API에서 401 응답을 받으면 /api/v1/auth/refresh로 access token을 재발급받고 요청을 재시도합니다.

Portfolio와 Project 관리

  1. 사용자는 /home에서 Portfolio 목록을 확인합니다.
  2. Portfolio 생성, 수정, 삭제는 /api/v1/portfolios API를 사용합니다.
  3. 특정 Portfolio 안의 Project 목록과 상세는 /api/v1/projects API를 사용합니다.
  4. 클라이언트 라우트 파라미터는 :portfolioCode와 :projectCode를 사용합니다.

공개 페이지

  1. 외부 사용자는 /public/:username/:portfolioCode로 공개 Project 목록에 접근합니다.
  2. Project 상세는 /public/:username/:portfolioCode/:projectCode로 접근합니다.
  3. 공개 API는 인증 헤더 없이 호출되며 공개 상태인 Portfolio와 Project만 노출해야 합니다.
