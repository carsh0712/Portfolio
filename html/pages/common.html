Architecture

연동 구조

React SPA가 화면 라우팅과 사용자 입력을 처리합니다.
API 클라이언트인 src/utils/api.ts가 VITE_API_BASE_URL 또는 같은 origin의 /api/v1/* 경로로 요청합니다.
인증 요청에는 Authorization: Bearer {access_token} 헤더가 붙습니다.
Flask 서버는 /api/v1/* REST API를 Blueprint로 제공합니다.
MySQL과 uploads에는 도메인 데이터와 실제 파일이 저장됩니다.
Client SPA
  -> src/utils/api.ts
  -> /api/v1/*
  -> Flask app.py
  -> routers / schemas / models
  -> MySQL + uploads directory
Flows

주요 흐름

로그인과 토큰 갱신

  1. 사용자가 /login에서 email과 password를 입력합니다.
  2. 클라이언트가 POST /api/v1/auth/login을 호출합니다.
  3. 성공 시 access token과 refresh token을 localStorage.auth_tokens에 저장합니다.
  4. 인증 API에서 401을 받으면 POST /api/v1/auth/refresh로 access token을 재발급받고 요청을 재시도합니다.

Portfolio와 Project 관리

  1. /home에서 로그인 사용자의 Portfolio 목록을 봅니다.
  2. /portfolio/:portfolioCode에서 해당 Portfolio의 Project 목록을 봅니다.
  3. Project 상세 라우트는 :projectCode를 사용합니다. 실제 값은 Project id가 아니라 code입니다.
  4. 목록 검색은 태그 입력과 선택 태그를 기반으로 서버의 search query parameter를 활용합니다.

다중 프로필 관리와 Portfolio 적용

  1. 사용자는 프로필 관리 화면에서 여러 Profile을 생성, 수정, 삭제할 수 있습니다.
  2. Profile은 표시 이름, 한 줄 소개, 상세 소개, 대표 이미지, 외부 링크, 연락 채널처럼 공개 페이지에 노출될 자기소개 정보를 가집니다.
  3. Portfolio 생성 또는 수정 시 사용자는 자신의 Profile 중 하나를 선택해 연결합니다.
  4. Portfolio 하나에는 Profile 하나만 연결합니다. 연결하지 않으면 사용자 기본 정보 또는 기본 Profile을 fallback으로 사용할 수 있습니다.
  5. 공개 Portfolio 페이지는 연결된 Profile을 기준으로 작성자 영역, 공유 미리보기, 연락 링크를 표시합니다.

공개 페이지

  1. /public/:username/:portfolioCode는 인증 없이 공개 Project 목록을 조회합니다.
  2. /public/:username/:portfolioCode/:projectCode는 인증 없이 공개 Project 상세를 조회합니다.
  3. 공개 API는 공개 상태인 Portfolio와 Project만 반환해야 합니다.
