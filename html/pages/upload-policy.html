이미지 업로드 정책

포트폴리오 이미지와 프로젝트 스크린샷 업로드 기준을 운영자가 빠르게 확인할 수 있도록 정리한 문서입니다.

업데이트: 2026-07-07 원본 문서: docs/upload-policy.md
요약 공통 정보 클라이언트 서버 SEO 업로드 정책 프로필 기획 개선 상태 Note
Summary

운영 기준 요약

허용 이미지

jpg, jpeg, png, webp만 허용합니다.

MIME 기준은 image/jpeg, image/png, image/webp입니다.

차단 이미지

gif는 업로드할 수 없습니다.

GIF는 리사이즈 및 WebP 변환 대상이 아니므로 운영 정책상 차단합니다.

최대 크기

MAX_FILE_SIZE=10485760

기본 운영 기준은 10 MB입니다.

Formats

허용 형식

구분허용 값
확장자jpg, jpeg, png, webp
MIME 타입image/jpeg, image/png, image/webp
차단gif, image/gif
참고: 파일 업로드 API 자체는 문서와 압축 파일도 받을 수 있지만, 포트폴리오 이미지와 프로젝트 스크린샷 업로드 UI는 위 이미지 형식만 허용합니다.
Resize

변환 및 리사이즈

산출물긴 변 제한WebP 품질파일명용도
상세 이미지 2048 px 이하 82 {user_id}-{uuid}.webp 기본 조회, 상세 화면
썸네일 480 px 이하 78 {user_id}-{uuid}.thumb.webp ?variant=thumbnail, 목록/미리보기

서버는 EXIF 방향 정보를 반영하고, Pillow WebP 저장 옵션 method=6을 사용합니다.

Storage

원본 보존 정책

운영 결정: 이미지 원본 바이너리는 별도로 보존하지 않습니다. 변환에 성공한 이미지는 상세 WebP와 썸네일 WebP만 저장합니다.
  • 데이터베이스에는 사용자가 올린 원본 파일명인 original_filename을 메타데이터로 보존합니다.
  • 다운로드 응답의 download_name은 원본 파일명을 사용합니다.
  • 실제 저장 파일은 변환된 WebP 파일입니다.
Failure

실패 및 예외 기준

  • 허용 확장자가 아니면 서버에서 400 응답으로 거절합니다.
  • GIF는 클라이언트 선택 단계와 업로드 요청 전에 차단하고, 서버에서도 허용하지 않습니다.
  • MAX_FILE_SIZE를 초과하면 서버에서 400 응답으로 거절합니다.
  • 썸네일 파일이 존재하면 thumbnail 변형으로 제공합니다.
  • 파일 삭제 시 상세 파일과 썸네일 파일을 함께 삭제합니다.
