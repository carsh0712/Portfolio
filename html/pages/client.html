Client

개요

역할

React 기반 Portfolio 관리 SPA입니다. 로그인 후 Portfolio와 Project를 생성, 수정, 조회하고 공개 URL을 복사해 외부에 공유할 수 있습니다.

기술 스택

  • React 19, react-router-dom 7
  • Vite 7, @vitejs/plugin-react-swc
  • Tailwind CSS 4
  • Vitest, Testing Library, MSW, Playwright
Structure

디렉터리 구조

경로역할
src/App.tsxReact Router 구성입니다. 인증 페이지, 공개 페이지, 보호 페이지를 분리합니다.
src/pages화면 조립을 담당합니다. 최근 PortfolioList, ProjectList, ProjectDetail은 로직을 훅으로 분리하고 JSX 중심으로 정리했습니다.
src/hooks페이지 상태/API 로직과 필터 로직을 담당합니다. usePortfolioListPage, useProjectListPage, useProjectDetailPage, useProjectFilters가 있습니다.
src/components재사용 UI입니다. 폼 필드, 에러, 카드, 뒤로가기, Project 상세/편집/필터/라이트박스 컴포넌트가 분리되어 있습니다.
src/typesauth.ts, portfolio.ts, project.ts 타입 정의입니다.
src/utils/api.ts서버 API 호출 계층입니다. access token 자동 첨부와 refresh token 기반 재시도를 처리합니다.
src/utils/projectMappers.ts서버 응답을 화면용 Project 타입으로 변환합니다.
src/test, *.test.tsxMSW 기반 API mock과 컴포넌트/유틸 테스트입니다.

Environment

클라이언트 .env 작성 방법

Vite 클라이언트에서 브라우저로 노출해야 하는 환경 변수는 반드시 VITE_ 접두사를 사용합니다. 현재 클라이언트는 src/utils/api.ts에서 VITE_API_BASE_URL을 읽어 Flask API 서버 주소로 사용합니다.

.env

모든 모드에서 공통으로 사용할 기본값입니다. 운영 빌드에 섞이지 않도록 기본값은 같은 origin을 쓰는 빈 값으로 둡니다.

VITE_API_BASE_URL=

.env.development

npm run dev처럼 development 모드에서 기본값을 덮어씁니다.

VITE_API_BASE_URL=http://localhost:8000

.env.production

npm run build는 production 모드로 실행되므로 .env.production 값을 함께 읽습니다. Flask가 같은 도메인에서 정적 파일과 API를 함께 제공하는 현재 구조에서는 빈 값으로 둡니다. 

VITE_API_BASE_URL=
  • Vite 개발 서버는 보통 http://localhost:5173에서 실행됩니다.
  • Flask 서버가 http://localhost:8000에서 실행되면 위 설정으로 API 요청이 정상 연결됩니다.
  • 운영 빌드에서 값이 http://localhost:8000이면 실제 사용자 브라우저가 자신의 PC localhost로 요청하므로 실패합니다.
  • API 서버가 별도 도메인이라면 https://api.example.com처럼 HTTPS origin을 지정하고 서버 CORS에도 해당 origin을 등록합니다.
  • .env, .env.development, .env.production은 로컬 설정 파일이므로 저장소에 커밋하지 않습니다.
  • 환경 변수 값을 바꾼 뒤에는 Vite 개발 서버를 재시작해야 반영됩니다.
  • 운영 환경 변수 값을 바꾼 뒤에는 반드시 다시 npm run build하고 새 dist를 배포해야 합니다.
