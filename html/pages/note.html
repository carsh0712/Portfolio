Client Performance

Lazy Import 정리

lazy import는 처음 앱을 열 때 모든 페이지 코드를 한 번에 내려받지 않고, 실제로 해당 페이지가 필요해졌을 때 JavaScript chunk를 나누어 불러오는 방식입니다.

왜 필요한가

  • 초기 로딩 시 내려받는 JavaScript 크기를 줄일 수 있습니다.
  • 사용자가 방문하지 않은 페이지 코드는 나중에 필요할 때만 로드됩니다.
  • Vite의 500kB 이상 chunk 경고를 줄이는 데 도움이 됩니다.

이번 적용

  • src/App.tsx의 페이지 import를 React.lazy()로 변경했습니다.
  • Suspense fallback으로 페이지 chunk 로딩 중 상태를 보여줍니다.
  • vite.config.ts에서 React 계열 라이브러리를 react-vendor chunk로 분리했습니다.

예시

import { lazy, Suspense } from 'react';

const ProjectDetail = lazy(() => import('./pages/ProjectDetail'));

function App() {
  return (
    <Suspense fallback={<PageState loading message="Loading page..." />}>
      <ProjectDetail />
    </Suspense>
  );
}
