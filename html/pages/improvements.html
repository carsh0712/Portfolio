  • 토큰 refresh 재시도 로직에서 깨진 주석 때문에 retryHeaders 선언이 주석 처리될 수 있던 문제를 고쳤습니다.
Client

남은 클라이언트 개선

  • 파일명과 컴포넌트명에는 아직 Category*가 남아 있습니다. 호환성 유지를 위해 이번 변경에서는 내부 타입/API부터 정리했습니다.
  • README가 기본 Vite 템플릿 설명에 가깝습니다. 실제 앱 실행법, 환경변수, API 서버 연결법을 보강하는 것이 좋습니다.
  • Project.links는 camelCase와 snake_case를 모두 허용하고 있어 응답 표준을 통일하면 변환 코드가 줄어듭니다.
Server

남은 서버 개선

  • portfolio_project_server_fastapi라는 이름과 README 제목은 실제 Flask 구현과 맞지 않습니다.
  • database.py에 기본 DB 비밀번호가 코드에 들어 있습니다. 개발 편의값이라도 환경변수 중심으로 정리하는 편이 안전합니다.
  • JWT_SECRET_KEY 기본값이 fallback-secret-change-me입니다. 운영 환경에서는 반드시 강한 시크릿을 환경변수로 설정해야 합니다.
  • 공개 파일 API는 username과 file_uuid 소유권만 확인합니다. 특정 파일이 실제 공개 portfolio/project에서 참조되는지까지 확인하지는 않습니다.
  • 검색은 JSON 필드를 문자열로 cast한 뒤 LIKE를 사용합니다. 규모가 커지면 MySQL JSON 검색 또는 별도 색인 전략이 필요합니다.
