이미지 업로드 최적화

  • JPG, PNG, WebP 업로드 시 서버에서 WebP 상세 이미지로 변환합니다.
  • 상세 이미지는 긴 변 2048px 이하로 제한합니다.
  • 썸네일은 긴 변 480px 이하의 .thumb.webp 파일로 별도 생성합니다.
  • ?variant=thumbnail 쿼리로 썸네일을 조회할 수 있습니다.
  • 파일 삭제와 포트폴리오/프로젝트 정리 시 썸네일도 함께 삭제합니다.

이미지 업로드 정책 문서화

  • docs/upload-policy.md 내용을 HTML 페이지로도 확인할 수 있게 했습니다.
  • 허용 이미지는 JPG, PNG, WebP로 제한하고 GIF는 업로드 차단 대상으로 명시했습니다.
  • 최대 크기, WebP 변환 품질, 상세/썸네일 크기, 원본 미보존 정책을 문서화했습니다.

클라이언트 이미지 사용 분리

  • 목록 카드, 업로드 미리보기, 상세 화면의 작은 스크린샷은 썸네일을 사용합니다.
  • 라이트박스와 크게 보는 화면은 상세 이미지를 사용합니다.
  • 공개 페이지 이미지 URL도 thumbnail과 detail 변형을 구분합니다.

Flask에서 빌드 클라이언트 제공

  • portfolio_project_client_vite/dist/index.html이 있으면 http://localhost:8000/에서 클라이언트를 제공합니다.
  • CLIENT_DIST_DIR 환경변수로 빌드 클라이언트 경로를 오버라이드할 수 있습니다.
  • /assets/*, /red_potion.png 같은 빌드 산출물도 Flask가 서빙합니다.
  • /public/admin/MYAPPS 같은 SPA 직접 접근 경로는 index.html로 fallback 됩니다.
  • /api/v1/*, /docs, /apispec.json은 API/Swagger 경로로 유지합니다.

API base와 CORS 정리

  • 서버에서 같이 제공되는 빌드 클라이언트는 기본적으로 같은 origin의 /api/v1/*를 호출합니다.
  • .env.production에 운영 API 도메인이 고정되어 빌드되는 문제를 제거했습니다.
  • 같은 origin 요청은 CORS DB 등록 없이 허용합니다.
  • 외부 origin은 기존처럼 cors_origin 테이블 등록 여부를 기준으로 허용합니다.

보안 기본값 정리

  • 추적 파일에 남아 있던 DB 비밀번호 기본값 dsc111을 제거했습니다.
  • MYSQL_PASSWORD가 없으면 서버가 명확한 에러 메시지와 함께 종료됩니다.
  • .env, .env.*, dist, uploads, logs는 Git ignore 대상으로 확인했습니다.

운영 배포 환경 체크

  • scripts/check-production-env.mjs를 추가했습니다.
  • DB, JWT, 업로드 경로, 클라이언트 빌드 경로, DB 재시도 설정 누락을 배포 전에 확인합니다.
  • --cors-origin으로 운영 공개 origin 지정 여부를 확인합니다.
  • 검사 성공 시 cors_origin 등록 SQL을 출력합니다.

문서와 테스트 보강

  • 서버 문서에 이미지 변형 API, 클라이언트 빌드 제공 방식, 실행 방법을 반영했습니다.
  • 이미지 리사이즈와 썸네일 생성 테스트를 추가했습니다.
  • SPA fallback과 같은-origin CORS 테스트를 추가했습니다.
  • 프론트 테스트와 빌드 검증을 통과했습니다.
