Completed

반영 완료

서버 프로젝트명 정리: 실제 구현이 Flask 기반인 점에 맞춰 서버 디렉터리와 문서를 portfolio_project_server_flask 기준으로 정리했습니다.
  • 상위 묶음 도메인을 Portfolio, 개별 작업물을 Project로 통일했습니다.
  • 클라이언트 라우트 파라미터는 Project id가 아니라 code를 의미하므로 :projectCode 기준으로 정리했습니다.
  • 반복 UI를 AuthTextField, FormTextField, FormTextarea, FormError, FormActions, PageCard, BackLink 등으로 분리했습니다.
  • Project 상세/편집/필터/라이트박스 UI를 ProjectDetailView, ProjectLinkEditor, ProjectScreenshotEditor, ProjectFilterBar, ProjectImageLightbox 등으로 분리했습니다.
  • PortfolioList, ProjectList, ProjectDetail의 상태/API/라우팅 로직을 각각 usePortfolioListPage, useProjectListPage, useProjectDetailPage 훅으로 분리했습니다.
  • 서버 응답을 화면용 Project로 변환하는 src/utils/projectMappers.ts를 추가했습니다.
  • HTML 문서를 한 파일에 몰지 않고 index.html 진입점과 pages 하위 문서로 분리했습니다.
Remaining

남은 개선 후보

  • 일부 기존 TypeScript/Python 코드와 테스트 설명에는 아직 깨진 한국어 문자열이 남아 있습니다. 사용자에게 노출되는 문구부터 우선 복구하는 것이 좋습니다.
  • 공개 파일 API는 username과 file UUID의 소유자는 확인하지만, 해당 파일이 실제 공개 Portfolio 또는 Project에서 참조되는지까지 검증하는 방향을 추가로 검토할 수 있습니다.
  • Project 검색은 JSON 필드를 문자열로 cast해 LIKE 검색합니다. 데이터가 커지면 별도 검색 인덱스나 정규화 테이블을 검토하는 것이 좋습니다.
  • 클라이언트 번들 크기가 Vite 기준 500kB를 넘는 경고가 있습니다. 라우트 단위 lazy loading과 chunk 분리를 검토할 수 있습니다.
  • 페이지 로직 훅에 대한 단위 테스트를 추가하면 렌더링 테스트 없이 API/상태 전이를 더 쉽게 검증할 수 있습니다.
