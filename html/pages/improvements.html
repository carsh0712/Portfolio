개선 사항

정적 코드 분석과 문서화 과정에서 확인한 개선 내용과 현재 반영 상태입니다.

요약 공통 정보 클라이언트 서버 개선 사항
Completed

진행 완료

도메인 용어 통일: 서버와 클라이언트의 상위 묶음 개념을 Portfolio로 통일하고, 개별 작업물은 Project로 분리했습니다.
  • 클라이언트 파일명과 컴포넌트명을 PortfolioList, PortfolioAdd, PortfolioEdit, PortfolioForm으로 변경했습니다.
  • 기존 상위 묶음 API alias를 제거하고 getPortfolios, createPortfolio, updatePortfolio, deletePortfolio를 사용하도록 정리했습니다.
  • Project 관련 API는 getProjects, getProjectDetail, updateProject로 분리했습니다.
  • 클라이언트 라우트 파라미터는 :portfolioCode와 :projectCode를 사용합니다.
  • Project의 상위 묶음 참조 필드를 Project.portfolioCode로 변경했습니다.
  • HTML 문서의 공통 도메인, 클라이언트 라우트, API 함수 표도 새 용어에 맞춰 갱신했습니다.
Remaining

남은 개선 후보

  • 일부 테스트와 화면 문자열의 한국어가 깨진 상태로 남아 있습니다. 사용자에게 노출되는 문구는 UTF-8 기준으로 별도 복구가 필요합니다.
  • portfolio_project_server_fastapi 이름과 실제 구현 프레임워크가 일치하는지 확인하고 README와 실행 문서를 계속 맞춰야 합니다.
  • 공개 파일 API는 username과 file UUID뿐 아니라 실제 공개 Portfolio 또는 Project에서 참조되는 파일인지까지 검증하는 방향을 검토할 수 있습니다.
  • Project 검색이 커지면 JSON 필드 문자열 검색 대신 DB 인덱스나 별도 검색 전략을 검토하는 것이 좋습니다.
